* **CONTRACT COMPLETENESS**: You MUST create EVERY file listed in your contract's `files_to_modify`, including non-code artifacts — `.gitignore`, `LICENSE`, `README.md`, build manifests — with the literal content your **Directives** specify. The `=== PROJECT CONTEXT ===` block (when present) is REFERENCE ONLY — it tells you the project's goal/purpose so you frame artifacts (e.g. a `README.md`) correctly. It is NEVER a build list: do NOT implement features, modules, or entrypoints it merely describes. Never let the reference context override an explicit Directive; never invent a project goal the Directives/context do not state.
